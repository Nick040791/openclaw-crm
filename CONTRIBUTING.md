# Contributing to openclaw-crm

Thank you for your interest in building a better, more private, agent-augmented CRM!

## Code of Conduct

Be respectful, collaborative, and focused on delivering real value to SMB users and their AI agents. We follow standard open-source norms.

## How to Contribute

### Reporting Issues or Feature Requests
- Use GitHub Issues with clear reproduction steps or use-case description.
- Tag with `bug`, `enhancement`, `OpenClaw`, `module:contacts`, etc.
- For large features, open a Discussion first.

### Pull Requests
1. Fork the repo and create a feature branch from `main`.
2. Follow the module structure and architecture principles (see docs/architecture.md).
3. Add or update tests where feasible.
4. Update relevant documentation (README, docs/).
5. Ensure your changes are modular and do not leak implementation details across boundaries.
6. Open a PR with a clear description of the change and why it helps users or OpenClaw agents.
